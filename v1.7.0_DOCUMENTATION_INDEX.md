## Release Contents

### Phase 18.4 Modules (3 New)

#### 1. Special Orbits (`pytcl/astronomical/special_orbits.py`)
**Purpose:** Parabolic and hyperbolic orbital mechanics
- **Size:** 537 lines of code
- **Functions:** 14 public + 6 helper functions  
- **Tests:** 31/31 passing ✅
- **Type Coverage:** 100% (mypy strict mode)
- **Quality:** 0 flake8 violations

**Key Functions:**
- `classify_orbit()` - Determine orbit type by eccentricity
- `mean_to_parabolic_anomaly()` - Parabolic anomaly solver
- `parabolic_anomaly_to_true_anomaly()` - Convert parabolic to true anomaly
- `hyperbolic_anomaly_to_true_anomaly()` - Convert hyperbolic to true anomaly
- `escape_velocity_at_radius()` - Escape velocity calculator
- `hyperbolic_excess_velocity()` - Excess velocity for escape orbits

**Mathematical References:**
- Vallado et al. (2013) - Orbital Mechanics
- Battin (1999) - An Introduction to the Mathematics of Ballistic Trajectories

#### 2. N-Dimensional Assignment (`pytcl/assignment_algorithms/nd_assignment.py`)
**Purpose:** Extend assignment solving to 4D and higher dimensions
- **Size:** 379 lines of code
- **Functions:** 5 algorithms + 1 validation function
- **Tests:** 30/30 passing ✅
- **Type Coverage:** 100% (mypy strict mode)
- **Quality:** 0 flake8 violations

**Algorithms:**
- `greedy_assignment_nd()` - O(n log n) greedy heuristic
- `relaxation_assignment_nd()` - Lagrangian dual optimization
- `auction_assignment_nd()` - Price-based auction algorithm
- `detect_dimension_conflicts()` - Constraint violation detection
- `validate_cost_tensor()` - Input validation

**Performance Characteristics:**
| Algorithm | Complexity | Best For |
|-----------|-----------|----------|
| Greedy | O(n log n) | Quick heuristics |
| Relaxation | O(n³) per iter | Medium problems |
| Auction | O(n²m) | Dense problems |

#### 3. Network Flow (`pytcl/assignment_algorithms/network_flow.py`)
**Purpose:** Min-cost flow formulation and solving
- **Size:** 362 lines of code
- **Functions:** 4 public + 3 helper functions
- **Tests:** 5/5 active passing ✅ (13 skipped for Phase 19)
- **Type Coverage:** 100% (mypy strict mode)
- **Quality:** 0 flake8 violations

**Algorithms:**
- `min_cost_flow_successive_shortest_paths()` - Bellman-Ford solver
- `assignment_to_flow_network()` - Network construction
- `assignment_from_flow_solution()` - Extract assignments
- `min_cost_assignment_via_flow()` - High-level wrapper

**Performance Note:** O(VE²) complexity. Phase 19 will optimize to network simplex.
