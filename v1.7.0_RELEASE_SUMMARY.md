## Known Limitations

### Network Flow Solver
The Bellman-Ford successive shortest paths implementation is suitable for prototyping and validation but exhibits O(VE) complexity. For production use with large cost matrices (>100×100):
- **Recommendation:** Use relaxation or auction algorithms from `nd_assignment` instead
- **Phase 19:** Min-cost flow solver will be optimized with network simplex algorithm

### Performance Characteristics
| Algorithm | Complexity | Best For |
|-----------|-----------|----------|
| Greedy Assignment | O(n log n) | Quick heuristics |
| Relaxation | O(n³) per iteration | Medium problems |
| Auction | O(n²m) | Dense problems |
| Network Flow | O(VE²) current | Small problems (Phase 19 optimization planned) |
